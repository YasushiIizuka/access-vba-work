# inbox（客先 → こちら へのコード受け渡し場所）

客先の Access に入っている**現行コード**をこちらに伝えるための置き場です。
（こちら → 客先 は `src/` を使う。方向でフォルダが分かれています）

## 客先PCでの手順（ブラウザだけで完結）

1. VBE で対象モジュールのコードを**全選択してコピー**
2. github.com でこのリポジトリを開き、`inbox` フォルダに入る
3. 右上の **Add file → Create new file** をクリック
4. ファイル名を入力（下記の命名ルール参照）
5. 本文にコードを貼り付け
6. **Commit changes** を押す（コミットメッセージはそのままでOK、
   「Commit directly to the main branch」のままでOK）

既にあるファイルを直したいときは、ファイルを開いて**鉛筆アイコン（Edit）**で
編集 → Commit changes。

## 命名ルール

```
サービスセンター分: inbox/モジュール名.bas
債権管理課分:       inbox/債権管理課/モジュール名.bas
例: inbox/mod取込処理.bas
    inbox/債権管理課/mod督促処理.bas
```

- ファイル名の入力欄に `債権管理課/mod〜.bas` のように **スラッシュ区切りで
  入力するとフォルダに入ります**（GitHub の仕様）
- どの Access のものか分かるようにしたい場合は先頭に付ける
  （例: `inbox/取込_mod取込処理.bas`）

## 貼り付け前の注意（重要）

コードの中に**パスワード・接続文字列**が書かれていないか、ざっと目視確認する。
入っていたら、その行だけ `<<パスワード>>` のように置き換えてから貼る。

## こちら側の扱い

- 受け取ったコードは内容確認・修正の材料として使い、成果は `src/` に反映する
- 反映が済んだ inbox のファイルは削除してよい（Git の履歴には残るので消えない）
