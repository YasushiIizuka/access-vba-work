# 決定事項

仕様・設計の決定をここに記録する（日付＋決定内容＋理由）。

## 2026-07-17 運用方式の決定（構築時の引き継ぎ文書より）

| 論点 | 決定 | 理由 |
|------|------|------|
| コードの置き場所 | GitHub private リポジトリ | コピーボタン標準装備・履歴管理・アクセス制御が揃っている |
| Vercel 同期 | やらない | Vercel の URL はデフォルト公開。客先コードが認証なしで露出するリスクがある |
| Gist | 使わない | secret gist は「URL を知っていれば誰でも見える」ため機密性が不十分 |
| リポジトリ分離 | 客先ごとに1リポジトリ | 機密の分離のため。他プロジェクトとは完全に別管理 |
| リポジトリ名 | 客先が特定できない汎用名（access-vba-work） | 客先名の露出防止 |
