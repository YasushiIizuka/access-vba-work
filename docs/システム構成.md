# システム構成

客先 Access システム（WORCS）の全体構成。開発時はこの構成を前提にする。

## データベース分割構成（4ファイル）

| ファイル | 役割 |
|---------|------|
| WORCS_データ | データ本体（テーブル）。全データがここに貯まる |
| WORCS_取込 | 基幹システムから出力されたデータの取込処理 |
| WORCS_管理 | 管理系機能（詳細は未確認） |
| WORCS_利用者 | 利用者向けの処理・操作（フォーム・帳票等の想定） |

## データの流れ

```
基幹システム（出力）
      ↓
WORCS_取込（取り込み）
      ↓
WORCS_データ（蓄積）
      ↓
WORCS_利用者（処理・利用）
```

- バックエンド（WORCS_データ）にテーブルを集約し、フロントエンド各ファイルからリンクテーブル等で参照する、典型的な Access 分割構成と想定

## 未確認事項

- WORCS_管理 の具体的な役割・機能
- 各フロントエンドから WORCS_データ への接続方式（リンクテーブルか、DAO/ADO で直接開くか）
- 基幹システムからの出力ファイル形式（CSV / 固定長 / Excel 等）と受け渡し場所
- 32bit / 64bit Office の別（Declare 文の書き方に影響）
