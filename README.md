# access-vba-work

客先の Access VBA 開発用コード置き場（private リポジトリ）。

## 対象部署

| 部署 | こちら→客先 | 客先→こちら | 備考 |
|------|------------|------------|------|
| サービスセンター | `src/` 直下 | `inbox/` 直下 | WORCS システム（既存分） |
| 債権管理課 | `src/債権管理課/` | `inbox/債権管理課/` | 2026-07-21 追加 |

## 運用ルールの要約

- 自宅・事務所で Claude Code に VBA を書かせて `src/` に保存し、push する
- 客先ではブラウザで github.com を開き、該当ファイルのコピーボタン（Raw 表示右上）でコピー → Access VBE に貼り付ける
- **機密情報（接続文字列・パスワード・社内サーバーパス・客先名等）はコードに書かない**。必要な箇所はプレースホルダ（例: `<<サーバーパス>>`）にして客先で手入力する
- **Access のバイナリ（.accdb / .laccdb / .mdb）は絶対にコミットしない**（.gitignore で除外済み）。管理するのはコードのテキストのみ
- 1モジュール = 1ファイル。ファイル冒頭コメントに貼り付け先モジュール名を明記する
- 客先で判明した修正点・エラーは GitHub Issue にメモしておくと、戻ってから拾いやすい

## フォルダ構成

```
├── README.md          # 本ファイル
├── CLAUDE.md          # Claude Code 向けルール
├── .gitignore         # Access バイナリの除外
├── docs/
│   ├── 決定事項.md    # 仕様・設計の決定を記録
│   └── システム構成.md # サービスセンター（WORCS）の構成
├── 作業ログ.md        # セッションごとの作業記録
├── 現状まとめ.md      # 現状サマリー
├── inbox/             # 客先→こちら の受け渡し（直下=サービスセンター）
│   └── 債権管理課/    # 債権管理課の受け渡し
└── src/               # こちら→客先（直下=サービスセンター）
    ├── modXxx.bas     # 標準モジュール
    ├── clsXxx.cls     # クラスモジュール
    └── 債権管理課/    # 債権管理課のコード
```

## 日常の運用フロー

1. 自宅・事務所: Claude Code に依頼 → `src/` にファイル生成 → push
2. 客先: github.com でファイルを開く → コピー → VBE に貼り付け → 動作確認
3. 修正点は Issue かメモに残す
4. 自宅・事務所: 修正 → push → 次回訪問時に反映
