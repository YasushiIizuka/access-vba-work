  - 【環境構築】引き継ぎ文書に基づき作業環境を構築。GitHub private リポジトリ
    access-vba-work を作成し、c:\project\access-vba-work に clone、
    初期構成（README / CLAUDE.md / .gitignore / docs / ログ類 / src）を作成して push
  - 【構成把握】客先 Access システム（WORCS）の全体構成をヒアリングし
    docs/システム構成.md に記録。WORCS_データ（テーブル本体）／WORCS_取込（基幹

    WORCS_利用者（利用者処理）の4ファイル分割。
    流れ: 基幹出力 → 取込 → データに蓄積 → 利用者で処理
  - 【エラー70修正】WORCS_利用者を開いていると取込時のバックアップ作成
    （FileCopy srcDB, backupFile）がエラー70になる問題を診断・修正。
    原因は FileCopy がコピー元を排他で開く仕様。FSO.CopyFile への置き換え＋
    コピー後に DAO で開いて検証する modBackup.bas を作成 → 客先で動作確認OK
  - 【複数Excel一括取込】取込仕様（Excel → テンポラリ → 反映クエリで本番へ追加）
    に対し、週明け等の複数ファイルをまとめて取り込む modImportBatch.bas を作成。
    当初はファイル単位で反映まで行うループ案だったが、「反映は既存クエリで行うので
    テンポラリに複数ファイル分を積み上げるだけでよい」との指示で方式変更。
    最終形: FileDialog 複数選択 → テンポラリを最初に1回だけ空に →
    全ファイルを TransferSpreadsheet で順に追記 → 反映は既存処理に任せる
  - 【運用ルール変更】作業ログを1日1エントリ方式＋年次アーカイブに変更
    （大本の Dropbox\CLAUDE.md を更新。~/.claude/CLAUDE.md はそこへのシンボリックリンクと確認）
  - 【取込結果の行数表示】完了・中断メッセージを「ファイル数＋合計行数」の併記に変更
    （TransferSpreadsheet 前後の DCount 差分で行数を取得）。基幹側の出力件数との
    突き合わせ検算ができるように
  - 【push 自動化】このリポジトリに限り、指示なしで commit & push まで自動実行して
    よいとの承認を得た（プロジェクト CLAUDE.md に記録。他プロジェクトには適用しない）
  - 【運用反映の重複ダイアログ問題】inbox/運用反映.bas（客先の現行コード・inbox
    運用の初利用）を確認。反映ボタンの重複チェックで、登録済み1件ごとに MsgBox を
    出すため、1000件登録済みだと1000回ダイアログが出る問題。
    修正: 重複は黙ってスキップして件数を集計し、最後に「対象/追加/スキップ件数＋
    内訳先頭20件」を1回だけ表示する src/運用反映_修正版.bas を作成（モジュール全体
    差し替え）。コメントアウトされていた On Error GoTo Err_Handler も有効化
  - 【逆方向の受け渡し】客先→こちらのコード受け渡し用に inbox/ フォルダを新設。
    客先PCのブラウザで GitHub の Add file → Create new file に VBE のコードを
    貼り付けてコミットする運用（手順は inbox/README.md に記載、客先PCでも読める）。
    パスワード・接続文字列は貼る前に <<プレースホルダ>> へ置換する注意も明記
