# 現状まとめ

**最終更新日**: 2026-07-24

## 全体像
- 客先の Access VBA 開発を、GitHub private リポジトリ経由のコピペ運用で支援する環境
- **対象は2部署**:
  - SC（サービスセンター・WORCS）: src/SC/・inbox/SC/
  - SK（債権管理課）: src/SK/・inbox/SK/
- こちら→客先: Claude Code がコードを生成して src/ に push → 客先ブラウザ（github.com）でコピー → Access VBE に貼り付け
- 客先→こちら: 客先の現行コードを VBE からコピー → github.com のブラウザ編集で inbox/ に保存 → こちらで pull（手順は inbox/README.md）
- スクリーンショットも inbox 経由で受け渡しできる（2026-07-24 に実績）
- WORCS は4ファイル分割構成: WORCS_データ／WORCS_取込／WORCS_管理（中身なし）／WORCS_利用者。詳細は docs/システム構成.md

## 基本方針・判断軸
- 機密情報はコードに書かない（プレースホルダ方式）。Access バイナリはコミットしない
- 1モジュール = 1ファイル、コピペ単位で完結させる
- このリポジトリに限り push は指示不要で自動実行してよい（2026-07-17 承認）
- T_WORCS の登録日・更新日は Date（日付のみ）で統一（2026-07-17 決定）
- F_特定5講座 まわりの一覧の書式（色・並べ替え）は**コードで管理**する方針（2026-07-24 客先意向）
- **教訓**: フォームモジュールの「全文差し替え」指示は、先に inbox で現行コードを受領してから行う（2026-07-24 に既存コード消失事故）
