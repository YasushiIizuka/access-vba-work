# 現状まとめ

**最終更新日**: 2026-07-21

## 全体像
- 客先の Access VBA 開発を、GitHub private リポジトリ経由のコピペ運用で支援する環境
- **対象は2部署**（2026-07-21 に債権管理課を追加）:
  - サービスセンター（既存分・WORCS）: src/ 直下・inbox/ 直下
  - 債権管理課（新規）: src/債権管理課/・inbox/債権管理課/。システム内容はまだ未ヒアリング
- こちら→客先: Claude Code がコードを生成して src/ に push → 客先ブラウザ（github.com）でコピー → Access VBE に貼り付け
- 客先→こちら: 客先の現行コードを VBE からコピー → github.com のブラウザ編集で inbox/ に保存 → こちらで pull（手順は inbox/README.md）
- この往復運用は実際に機能することを確認済み（inbox 経由で運用反映.bas を受領し、修正版を客先で動作確認するまで一巡した）
- サービスセンターのシステム（WORCS）は4ファイル分割構成: WORCS_データ（テーブル本体）／WORCS_取込（基幹データ取込）／WORCS_管理（統計・ユーザー管理、中身はまだ無い）／WORCS_利用者（利用者処理）。詳細は docs/システム構成.md 参照

## 基本方針・判断軸
- 機密情報はコードに書かない（プレースホルダ方式）。inbox に貼る際もパスワード等は置換
- Access バイナリはコミットしない（テキストのみ管理）
- 1モジュール = 1ファイル、コピペ単位で完結させる
- PC 間の同期は GitHub の push / pull で行う（Dropbox 等の同期フォルダには置かない）
- このリポジトリに限り push は指示不要で自動実行してよい（2026-07-17 承認）
- T_WORCS の登録日・更新日は Date（日付のみ）で統一（2026-07-17 仕様決定）

  （遅くなったら複合インデックス＋Seek 化を検討）
